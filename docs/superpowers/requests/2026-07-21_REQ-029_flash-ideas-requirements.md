# 闪念（Flash Idea）功能 - 需求分析

**日期:** 2026-07-21
**状态:** 已定稿

## 概述
闪念功能用于快速记录日常思考中一闪而过的念头，以"培育森林"的概念来管理想法生命周期：一条闪念默认是"小树苗"，关联到任务后成长为"大树"，任务闭环后变成"森林"。

## 功能需求

### 1. 首页录入区
- **入口**：导航栏"我的文章"后面增加"闪念"入口
- **录入卡片**：Home 主体区新增独立闪念录入卡片
  - 标题"💡 闪念"
  - 文本域（默认 2 行，聚焦展开到 4 行）
  - 占位文字"此刻闪过什么念头？写下它..."
  - 提交按钮"记录闪念"，主色填充
  - 快捷键：Ctrl+Enter 提交，Enter 换行
- **最近闪念**：录入区下方展示最近 3 条闪念摘要，点击可跳转到闪念管理

### 2. 闪念管理页
- **入口**：后台管理 → 任务与订阅 → 闪念管理（第一条）
- **路由**：`/about/flash-ideas`
- **功能**：
  - 快速录入（复用首页录入组件）
  - 闪念列表：卡片布局，按创建时间倒序
  - 每条显示：内容文字、创建时间、状态标签、关联任务名称
  - 操作：编辑内容、关联任务、删除

### 3. 状态流转

| 状态 | 含义 | 条件 |
|------|------|------|
| 🌱 小树苗 (sapling) | 默认状态 | 新建时 |
| 🌳 大树 (tree) | 已关联任务 | 关联 task_id 时 |
| 🌲 森林 (forest) | 关联任务已完成 | 查询时自动检测 |

- 关联任务时自动变为 tree
- 查询列表时后端 JOIN tasks 表检测关联任务状态，已完成的自动标记为 forest

### 4. 数据模型
- 表名：`flash_ideas`
- 字段：id, content(TEXT), status(ENUM), task_id(INT NULL), created_at, updated_at
- 索引：status, task_id

### 5. API
- `GET /api/flash-ideas` — 列表（倒序，含任务信息）
- `POST /api/flash-ideas` — 新建
- `PUT /api/flash-ideas/:id` — 更新（content / task_id）
- `DELETE /api/flash-ideas/:id` — 删除

### 6. 非功能需求
- 所有新页面需包含移动端适配（@media max-width: 768px）
- 遵循项目深色主题风格
- 与现有任务系统关联（查询 tasks 表）
