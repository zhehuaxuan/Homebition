# API 响应规范

## 当前现状（待统一）

当前各模块使用不同的响应格式：

- **查看文章**: `{ code: 0, rows: [...], total: N }`
- **登录**: `{ code: 200, message: "...", token: "...", user: {...} }`
- **订阅列表**: `{ code: 200, list: [...] }`
- **任务列表**: `{ list: [...] }`（无外层 code）
- **标签列表**: `{ rows: [...] }`

## 目标规范

```
成功: { code: 0, data: {...}, message: "ok" }
列表: { code: 0, data: { list: [...], total: N }, message: "ok" }
错误: { code: <400|401|404|500>, message: "错误描述" }
```

## HTTP 状态码使用

| 场景 | Status | code |
|------|--------|------|
| 成功 | 200 | 0 |
| 参数错误 | 200 | 400 |
| 未认证 | 401 | 401 |
| 不存在 | 200 | 404 |
| 服务端错误 | 500 | 500 |
